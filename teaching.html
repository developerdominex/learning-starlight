This page teaches developers how to write .slmd files and generate ready-to-use Markdown.

1. Writing SLMD Files

Use SLMD tags literally to structure your content:

<big1>My Document Title</big1>
<big2>Section 1</big2>
<bold>Bold Text Here</bold>
<italic>Italic Text Here</italic>
<code>Inline code</code>
<color red>Red Text</color>
<link https://example.com>Example Link</link>

<ul>
<li>Item 1</li>
<li>Item 2</li>
<li>Item 3</li>
</ul>

<table>
<tr><td>Header 1</td><td>Header 2</td></tr>
<tr><td>Row 1, Col 1</td><td>Row 1, Col 2</td></tr>
</table>

2. Saving the File

Save your file with the extension .slmd, e.g., example.slmd

3. Converting SLMD to Markdown

Use Starlight MD Converter in a program.sl script:

import { convertFile } from "starlight-mdconverter"

sldeploy "=== SLMD to Markdown Converter ==="

let inputFile = ask("Enter path to your .slmd file:")
let outputFile = ask("Enter path for output .md file:")

convertFile(inputFile, outputFile)

sldeploy "Conversion complete! Markdown file saved."

4. Notes

  • Use <big1>, <big2>, <big3> for headings.
  • Use <bold>, <italic>, <code> for text formatting.
  • Use <color colorname> to indicate colored text.
  • Use <ul> and <li> for lists.
  • Use <table>, <tr>, <td> for tables.

After conversion, all these tags will turn into proper Markdown syntax in your output .md file.
